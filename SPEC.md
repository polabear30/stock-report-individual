# 개별 종목 AI 분석 리포트 — 명세서 (SPEC v1)

## 1. 목적 / 범위
- 5개 종목의 **개별 심층 AI 분석 리포트**를 매일 자동 생성 → GitHub Pages 게시.
- 디자인·구성은 첨부 `analysis_report_v2_9.html`를 **그대로 재현**.
- "전체 시황 리포트"(stock-report)와 **동일한 파이프라인**(정적 HTML → Pages → 매일 07:00 KST).
- **데이터**(시세·지표)는 계산하고, **판단**(카탈리스트·시나리오·전략·점수)은 LLM이 **그라운딩된 사실 기반**으로 생성.

## 2. 종목별 섹션 (첨부 재현)
요약박스(통합판단) · 최종결론 · 핵심 메트릭 그리드 +
1. 기술적 분석 · 일봉
2. 기술적 분석 · 120분봉
3. 호재 카탈리스트 / 리스크
4. 엘리어트 파동 / 시나리오 (+확률)
5. 트레이딩 전략 (매수/매도 트리거)
6. AI 종합 점수 (5개 하위 + 종합)
7. 손익 시뮬레이션
8. 확률 분포
9. 투자자별 전략
10. 시장 심리 / 애널리스트
11. PCR 옵션 분석

구성: **단일 페이지에 5종목 스택 + 상단 종목 바로가기**(첨부와 동일 형식). 반응형·모든 브라우저.

## 3. 데이터 vs AI 판단 경계
| 영역 | 항목 | 출처 |
|---|---|---|
| **계산(데이터)** | 시세·등락·거래량·52주, RSI·MACD·볼린저·스토캐스틱·이평(일봉+120분봉), 옵션 PCR, 애널리스트 목표가, 어닝일, 시총·PER 등 | yfinance + `ta` |
| **AI 판단** | 지표 해석, 카탈리스트, 리스크, 엘리어트 파동, 시나리오+확률, 트레이딩 전략, 종합점수, 손익/확률분포, 투자자별 전략, 결론 | LLM (그라운딩) |

## 4. 데이터 파이프라인
- **yfinance**: OHLCV(일봉; 60분봉 → **120분봉 리샘플**), `ta`로 지표 계산, `.info`(시총·PER·52주), `.recommendations`/`target`(애널리스트), `.calendar`(어닝일), `.option_chain`(PCR).
- **Alpha Vantage `NEWS_SENTIMENT`**: 종목별 최근(약 1~2주) 뉴스·감성 → **카탈리스트/리스크 그라운딩 근거**.
- 레버리지 ETF는 어닝/목표가 등 미적용(아래 7번).

## 5. AI 판단 (LLM) — provider 중립
- `llm/client.py` 추상화. **`LLM_PROVIDER=openai`(기본) | `anthropic`(폴백)** 환경변수 한 줄로 전환.
- **입력(그라운딩 컨텍스트)**: 계산된 지표 + 수집된 뉴스 기사(제목·요약·감성·날짜) + 애널리스트/어닝.
- **출력**: 엄격 JSON 스키마(OpenAI Structured Outputs / Anthropic tool) — `catalysts[]`, `risks[]`, `scenarios[]`, `scores{}`, `strategy{}`, `waves[]`, `conclusion`, `investor_strategy`, `sentiment` 등.
- 종목 유형(단일주 / 레버리지ETF)에 따라 **스키마·프롬프트 분기**.

## 6. ⚠ 할루시네이션 방지 (최우선 원칙)
- 카탈리스트·리스크·뉴스성 수치는 **반드시 제공된 뉴스 컨텍스트의 실제 기사에서만** 추출.
- 각 카탈리스트/리스크 항목에 **근거(기사 제목·날짜)** 연결.
- 근거 없으면 **"최근 특이 뉴스 없음"** 으로 표기하고 **지어내지 않음**.
- 시나리오 확률·엘리어트 파동·목표가는 본질적으로 **추정/주관** → 면책 문구 강화, "참고용·투자권유 아님" 유지.

## 7. 레버리지 ETF 변형 (SOXL · KORU)
- **제외**: 어닝, 애널리스트 목표가, 내부자매매, 펀더멘털/실적 점수.
- **대체/추가**: 기초지수(반도체/한국증시) 방향, **일일 리밸런싱·레버리지 소실(decay) 경고**, 변동성, 권장 보유기간 경고.
- 시나리오·전략은 기초지수·변동성 기반.

## 8. 전달 / 자동화
- 정적 HTML → **GitHub Pages**, 매일 **07:00 KST**(cron `0 22 * * *` UTC). 전체시황과 **별도 repo·Actions**.
- 데이터/LLM 실패 시 **재시도 + 완결성 표시**(전체시황 패턴 재사용). 일부 실패해도 말없이 넘어가지 않음.

## 9. 파일 구조 (신규 repo)
```
stock-individual/
  config.py                 # 종목 목록·유형(단일주/레버리지ETF)·설정
  fetch/
    market.py               # yfinance OHLCV·지표·info·애널리스트·어닝·옵션
    news.py                 # Alpha Vantage 뉴스/감성 (그라운딩)
  llm/
    client.py               # provider 추상화 (openai/anthropic 전환)
    schema.py               # 출력 JSON 스키마 (단일주/ETF)
    prompts.py              # 그라운딩 프롬프트 (날조 금지 원칙)
  render/
    template.py             # 첨부 디자인 HTML 템플릿
    sections.py             # 종목별 섹션 렌더
  generate.py               # 오케스트레이터: 종목별 fetch→llm→render→HTML
  .github/workflows/daily.yml
  requirements.txt
  SPEC.md
```

## 11. 비용 / 리스크
- LLM 비용: 5종목 × 매일 (OpenAI 프로젝트 예산 상한으로 통제).
